## CSS File Structure

### Current Architecture ✅ **EXCELLENT**

```
/styles/
├── globals.css              # Main entry point, Tailwind directives
├── themes.css               # Theme orchestration (imports all files)
├── themes-variables.css     # Theme-agnostic tokens (fonts, spacing)
├── themes-light.css         # Light mode colors
├── themes-dark.css          # Dark mode colors (.dark override)
└── utilities.css            # BEM component classes (twb-)
```

**Import Order (themes.css):**
1. `themes-variables.css` - Core tokens (fonts, spacing, shadows, radii)
2. `themes-light.css` - Light mode colors (default)
3. `themes-dark.css` - Dark mode colors (.dark class)
4. `utilities.css` - BEM component classes
5. `@theme inline` - Tailwind v4 mappings

## Critical Issues (1) - Fix Immediately

### 1. ❌ Hardcoded Hex Colors in TSX Files (CSS Variable Migration)

**Severity:** 🔴 **CRITICAL**  
**Guideline:** `/guidelines/development/wordpress-css-variables.md` (MANDATORY)  
**Impact:** Cannot manage theme consistently, violates design token standards

**Issue:** 20+ TSX files use hardcoded hex colors instead of CSS variables

**Examples:**
```tsx
// ❌ BAD - Hardcoded hex colors in TSX
```

```tsx
// ✅ GOOD - CSS variables
<div className="bg-[var(--twb-color-gold)]">
```

### 4. ⚠️ Inconsistent Dark Mode Variable Naming

**Severity:** 🟠 **HIGH**  
**Impact:** Confusing which variables change in dark mode

**Issue:** Some variables have dark mode overrides, some don't. No clear pattern.

**Current State:**
```css
/* themes-light.css */
:root {
  --twb-color-paper: #f5efe4; /* No dark override (stays light) */
  --twb-color-bg-primary: var(--twb-color-paper); /* Dark override */
}

/* themes-dark.css */
.dark {
  --twb-color-bg-primary: #1a1714; /* Overrides */
  /* --twb-color-paper NOT overridden */
}
```

**Confusion:** Is `--twb-color-paper` supposed to stay light in dark mode? Or is it a bug?

**Recommendation:**
```css
/* Brand colors (never change) */
--twb-brand-paper: #f5efe4;
--twb-brand-ink: #1e1a17;
--twb-brand-plum: #5a2d3b;

/* Semantic colors (change in dark mode) */
--twb-color-bg-primary: var(--twb-brand-paper); /* light */
.dark {
  --twb-color-bg-primary: var(--twb-brand-ink); /* dark */
}
```

**Remediation:**
- [ ] Separate brand colors (--twb-brand-*) from semantic colors (--twb-color-*)
- [ ] Document which variables have dark mode overrides
- [ ] Add comments in CSS files explaining logic
- [ ] Update dark mode guidelines

**Reference:** `/guidelines/design-tokens/dark-light-mode.md` ⚠️ **MANDATORY**
