## Appendix: Route Audit Methodology

**Scan Process:**
1. Read `/routes.tsx` and extract all registered routes
2. List all files in `/pages/` recursively
3. Read all data files in `/data/` and check for slugs/IDs
4. Search for all `<Link>`, `useNavigate`, `to=`, `href=` in components and pages
5. Cross-reference extracted links against route map
6. Calculate health score based on valid vs. broken links

**Validation Rules:**
- Internal links only (skip external URLs)
- Dynamic routes validated against data files
- Missing slugs flagged for auto-generation
- Broken links categorized by severity

**Slug Generation Rules:**
1. Convert to lowercase
2. Replace spaces with hyphens
3. Remove special characters
4. Remove multiple consecutive hyphens
5. Trim leading/trailing hyphens
6. Check uniqueness (append -2, -3 if duplicate)
