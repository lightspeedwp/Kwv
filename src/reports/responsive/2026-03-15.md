## Mobile Considerations

### Hero Height Strategy ✅ **EXCELLENT**

**Pattern:** `min-h-[calc(100dvh-90px)]` on mobile

**Why this is correct:**
- Uses `dvh` (dynamic viewport height) for mobile browsers
- Accounts for 90px sticky header
- Prevents content being hidden under mobile URL bars
- Desktop uses `md:min-h-screen` for full viewport

**Example:**
```tsx
// /components/sections/Hero.tsx
heightClass = {
  full: 'min-h-[calc(100dvh-90px)] md:min-h-screen', ✅
  large: 'min-h-[calc(100dvh-90px)] md:min-h-[80vh]', ✅
  medium: 'min-h-[calc(100dvh-90px)] md:min-h-[60vh]', ✅
  small: 'min-h-[300px] md:min-h-[40vh]', ✅
}
```

**Assessment:** ✅ **Perfect mobile viewport handling**

### 2. ❌ Fixed Widths Without Mobile Fallbacks

**Severity:** 🔴 **CRITICAL**  
**Impact:** Horizontal scroll on mobile devices

**Issue:** Some components may have fixed widths that overflow on small screens

**Pattern to search:**
```tsx
// ❌ Potential issue - fixed width, no responsive variant
<div className="w-[600px]"> // Overflows on mobile
  
// ✅ Good - max-width with 100% fallback
<div className="w-full max-w-[600px]">
  
// ✅ Good - responsive width
<div className="w-full md:w-1/2">
```

**Remediation:**
- [ ] Search for `w-[XXXpx]` without responsive variants
- [ ] Replace fixed widths with max-widths
- [ ] Test on 320px width (iPhone SE)
- [ ] Add `overflow-x-hidden` to body if needed

## High Priority Issues (5) - Fix Within Sprint

### 3. ⚠️ Tablet Breakpoint (768-1023px) Under-Utilized

**Severity:** 🟠 **HIGH**  
**Impact:** Sub-optimal experience on iPads/tablets

**Issue:** Most components jump from mobile (0-767px) to desktop (1024px+), skipping tablet (768-1023px)

**Pattern:**
```tsx
// ❌ Current - Mobile to Desktop jump
<div className="flex-col md:flex-row lg:grid-cols-3">
//               ^^^^ 768px  ^^^^ 1024px
// Tablet (768-1023px) gets desktop flex-row, might be too cramped

// ✅ Better - Progressive enhancement
<div className="flex-col md:flex-col lg:flex-row lg:grid-cols-3">
//               ^^^^ 768px still stacked  ^^^^ 1024px row layout
```

**Areas to review:**
- Grid layouts (2-col on tablet, 3-4 col on desktop)
- Navigation menus (tablet-specific layout?)
- Product cards (2 columns on tablet, 3-4 on desktop)
- Form layouts (stack on mobile/tablet, side-by-side on desktop)

**Remediation:**
- [ ] Audit all `md:` breakpoints
- [ ] Add tablet-specific layouts where needed
- [ ] Test on iPad (1024x768) and iPad Mini (768x1024)
- [ ] Consider `lg:` for desktop-only enhancements

### 4. ⚠️ No Landscape Mobile Orientation Support

**Severity:** 🟠 **HIGH**


**Remediation:**
- [ ] Test all pages in landscape mode
- [ ] Add `landscape:` utilities for hero sections
- [ ] Adjust modal heights for landscape
- [ ] Test on actual devices (not just browser resize)

### 5. ⚠️ Missing Responsive Images (srcset, sizes)

**Severity:** 🟠 **HIGH**  
**Impact:** Large images loaded on mobile (slow, wastes data)

**Issue:** Images use single `src`, no responsive image loading

**Current:**
```tsx
// ❌ Bad - One size for all
<img src="large-image-2000px.jpg" />
```

**Should be:**
```tsx
// ✅ Good - Responsive images
<img 
  src="large-1200px.jpg"
  srcSet="small-600px.jpg 600w, medium-900px.jpg 900w, large-1200px.jpg 1200w"
  sizes="(max-width: 768px) 100vw, (max-width: 1024px) 50vw, 33vw"
  alt="Product image"
/>
```

**Remediation:**
- [ ] Add srcset to product images
- [ ] Add srcset to hero images
- [ ] Use ImageWithFallback with responsive props
- [ ] Consider lazy loading (loading="lazy")
- [ ] Optimize image file sizes

---

### 6. ⚠️ Font Size Not Responsive Enough

**Severity:** 🟠 **HIGH**  
**Impact:** Text may be too small on mobile or too large on desktop

**Current State:**
```css
/* themes-variables.css - Fluid typography */
--twb-text-h1: clamp(2.4rem, 5vw + 1rem, 4.5rem); ✅
--twb-text-h2: clamp(2rem, 4vw + 1rem, 3.5rem); ✅
--twb-text-body: clamp(1rem, 1vw + 0.5rem, 1.125rem); ✅
```

**Assessment:** ✅ **Good fluid typography** with clamp()

**However:**
```tsx
// /components/ui/input.tsx
className="text-base md:text-sm" ✅ Good - larger touch targets on mobile
```

**This is actually correct:**
- Mobile: `text-base` (16px) - prevents iOS zoom on focus
- Desktop: `text-sm` (14px) - more compact

**Recommendation:** Continue this pattern for form inputs

**Remediation:**
- [ ] Verify all form inputs use `text-base` on mobile
- [ ] Test iOS zoom behavior (16px minimum prevents zoom)
- [ ] Document responsive font-size pattern
