### 3. ❌ Inconsistent Heading Hierarchy (WCAG 1.3.1)

**Severity:** 🔴 **CRITICAL**  
**WCAG:** 1.3.1 Info and Relationships  
**Impact:** Screen reader users cannot navigate page structure

**Issue:** Multiple pages skip heading levels (h1 → h3, missing h2)

**Examples Found:**
```tsx
// ❌ Bad - Skips h2
<h1>Welcome to Our Farm</h1>
<h3>Our Story</h3> {/* Should be h2 */}

// ❌ Bad - Multiple h1s
<h1>Page Title</h1>
<section>
  <h1>Section Title</h1> {/* Should be h2 */}
</section>
```

**WCAG Requirement:**
- Only ONE h1 per page
- Headings must be sequential (h1 → h2 → h3, no skipping)
- Use headings to create document outline

**Remediation:**
- [ ] Audit all pages for heading hierarchy
- [ ] Ensure one h1 per page
- [ ] No skipped levels (h1 → h2 → h3, not h1 → h3)
- [ ] Update Typography component with semantic defaults
- [ ] Test with screen reader outline view

**Files to audit:**
- All pages in `/pages/`
- All section components in `/components/sections/`

### 6. ⚠️ Color as Only Visual Cue (WCAG 1.4.1)

**Severity:** 🟠 **HIGH**  
**WCAG:** 1.4.1 Use of Color  
**Impact:** Colorblind users cannot distinguish states

**Issue:** Form validation, required fields, and interactive states rely only on color

**Examples:**
```tsx
// ❌ Bad - Red border only for errors
<input className={hasError ? 'border-red-500' : 'border-gray-300'} />

// ✅ Good - Icon + color + text
<input 
  className={hasError ? 'border-red-500' : 'border-gray-300'}
  aria-invalid={hasError}
  aria-describedby={hasError ? 'error-msg' : undefined}
/>
{hasError && (
  <div id="error-msg" className="text-red-500 flex items-center gap-1">
    <AlertCircle size={16} />
    <span>This field is required</span>
  </div>
)}

// ✅ Good - Asterisk for required fields
<label>
  Email address <span className="text-red-500" aria-label="required">*</span>
</label>
```

**Remediation:**
- [ ] Add icons to error states
- [ ] Include error text, not just red border
- [ ] Mark required fields with asterisk + aria-required
- [ ] Add text labels to color-coded UI (e.g., status badges)

**Files to audit:**
- All form components in `/components/shop/checkout/`
- `/components/shop/single-product/ProductAddToCart.tsx`
- Badge components with color variants

### 7. ⚠️ Focus Order Not Logical (WCAG 2.4.3)

```tsx
// ✅ Focus trap in modal
useEffect(() => {
  if (isOpen) {
    // Focus first interactive element

    // Trap focus within modal
    const handleKeyDown = (e: KeyboardEvent) => {
      if (e.key === 'Tab') {
        // Implement focus trap logic
      }
    };
    document.addEventListener('keydown', handleKeyDown);
    return () => document.removeEventListener('keydown', handleKeyDown);
  }
}, [isOpen]);
```

### 8. ⚠️ Missing ARIA Landmarks (WCAG 1.3.1)

**Severity:** 🟠 **HIGH**  
**WCAG:** 1.3.1 Info and Relationships  
**Impact:** Screen reader users cannot navigate by landmarks

**Issue:** Some pages missing semantic landmarks (banner, main, navigation, contentinfo)

**Current State:**
```tsx
// ❌ Missing landmark roles
<div> {/* Should be <header> or role="banner" */}
  <nav>...</nav>
</div>
<div> {/* Should be <main> or role="main" */}
  Page content
</div>
<div> {/* Should be <footer> or role="contentinfo" */}
  Footer content
</div>
```

**Required:**
```tsx
// ✅ Proper landmarks
<header> {/* Implicit role="banner" */}
  <nav aria-label="Main navigation">...</nav>
</header>
<main id="main-content"> {/* Implicit role="main" */}
  Page content
</main>
<footer> {/* Implicit role="contentinfo" */}
  Footer content
</footer>
```

**Remediation:**
- [ ] Verify Layout.tsx uses semantic HTML
- [ ] Add aria-label to navigation elements
- [ ] Ensure one main landmark per page
- [ ] Test with screen reader landmark navigation (NVDA: D key)
