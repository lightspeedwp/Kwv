## Executive Summary

Primary violators are shop/checkout forms and order status components that use `#333333`, `#111111`, `#D3D3D3`, `#8B0000`, and `#2C1810` hex colors directly in className attributes instead of CSS variables.

- ✅ **No inline style attributes** found (all violations in className)
- ⚠️ **Dark mode incompatible:** Hardcoded colors prevent dark mode support

**Recommendation:** Migrate all 96 hardcoded color values to CSS variables following WordPress-aligned token system. Estimated effort: 3-4 hours.

## Detailed Findings

### 1. **Checkout Form Components** (71 violations)

**Files Affected:**
1. `PickupLocationSelect.tsx` - 3 violations
2. `ShippingAddressForm.tsx` - 34 violations  
3. `BillingAddressForm.tsx` - 32 violations
4. `RadioButton.tsx` - 1 violation
5. `Checkbox.tsx` - 1 violation

**Common Hardcoded Colors:**
- `#333333` (dark gray text) - 60+ instances
- `#111111` (near-black text) - 10+ instances
- `#2C1810` (brown focus border) - 4 instances
- `#8B0000` (dark red radio checked) - 1 instance
- `#D3D3D3` (light gray borders) - Used in order components

**Example Violations:**

```tsx
// ❌ BAD: Hardcoded hex colors
<div className="border border-[#333333] p-4 rounded-sm">
  <span className="text-[#333333] text-lg">FREE</span>
</div>

<p className="text-[19px] text-[#111111] mb-6">
  Enter your shipping address
</p>

<input 
  className="focus:border-[#2C1810] text-[#333333]"
/>

<div className="peer-checked:border-[#8B0000]"></div>
```

**Should Be:**

```tsx
// ✅ GOOD: CSS variables
<div className="border border-[var(--twb-border-primary)] p-4 rounded-sm">
  <span className="text-[var(--twb-color-text-primary)] text-lg">FREE</span>
</div>

<p className="text-[var(--twb-font-size-body)] text-[var(--twb-color-text-primary)] mb-6">
  Enter your shipping address
</p>

<input 
  className="focus:border-[var(--twb-color-plum)] text-[var(--twb-color-text-primary)]"
/>

<div className="peer-checked:border-[var(--twb-color-plum)]"></div>
```

| `gray-300` | Border (Tailwind) | `var(--twb-border-primary)` | Form field borders |

**Note:** All colors must support both light and dark modes via CSS variables.
